# 猜诗词游戏需求规格说明

## 1. 概述

### 1.1 项目描述
将基于C++和Qt6开发的"猜诗词游戏"转换为ArkTS语言实现，运行于HarmonyOS平台。游戏采用Wordle式颜色反馈机制，玩家需要根据提示猜测正确的诗句。

### 1.2 技术栈
- **开发语言**: ArkTS
- **目标平台**: HarmonyOS
- **UI框架**: ArkUI声明式开发范式

### 1.3 核心价值
- 传承中华诗词文化，寓教于乐
- 提供直观的颜色反馈机制，增强游戏体验
- 支持题库和诗词库管理，实现内容可扩展

---

## 2. 功能需求

### 2.1 游戏核心功能

#### 2.1.1 题目生成
**需求描述**: 系统从题库中随机挑选一句诗作为答案，并提供上下句字数提示。

**验收标准**:
- **[EARS-001]** 当玩家开始新游戏时，系统应从题库中随机选择一道题目。
- **[EARS-002]** 当题目选定后，系统应显示该诗句的上下句字数提示。
- **[EARS-003]** 当题库为空时，系统应提示"题库为空，请先添加题目"。

#### 2.1.2 猜测输入
**需求描述**: 玩家输入猜测的诗句并提交。

**验收标准**:
- **[EARS-004]** 当玩家输入诗句并点击"猜测"按钮时，系统应接收并处理输入。
- **[EARS-005]** 当玩家输入的诗句包含标点符号时，系统应自动过滤标点后再进行匹配。
- **[EARS-006]** 当玩家输入为空时，系统应提示"请输入诗句"。

#### 2.1.3 匹配判断
**需求描述**: 系统判断玩家输入是否与答案匹配。

**验收标准**:
- **[EARS-007]** 当玩家输入的诗句与答案完全匹配时，系统应判定为游戏成功。
- **[EARS-008]** 当玩家输入的诗句字数与答案字数不一致时，系统应提示"请输入一句字数正确的古诗词"。
- **[EARS-009]** 当玩家输入的诗句字数正确但不在诗词库中时，系统应提示"请输入一句字数正确的古诗词"。

#### 2.1.4 颜色反馈机制
**需求描述**: 当诗句在诗词库中且字数正确时，系统使用颜色标记每个字的匹配状态。

**验收标准**:
- **[EARS-010]** 当答案中包含该字且位置正确时，系统应将该字标记为绿色。
- **[EARS-011]** 当答案中包含该字但位置不正确时，系统应将该字标记为黄色。
- **[EARS-012]** 当答案中不包含该字时，系统应将该字标记为灰色。
- **[EARS-013]** 当同一字在答案中出现多次时，系统应优先匹配位置正确的字，再匹配位置不正确的字。

#### 2.1.5 猜测次数限制
**需求描述**: 玩家每轮游戏有有限的猜测机会。

**验收标准**:
- **[EARS-014]** 当游戏开始时，系统应初始化剩余猜测次数为20次。
- **[EARS-015]** 当玩家每次提交猜测后，系统应将剩余猜测次数减1。
- **[EARS-016]** 当剩余猜测次数为0且未猜中时，系统应判定为游戏失败并显示正确答案。

#### 2.1.6 游戏状态显示
**需求描述**: 系统实时显示游戏状态信息。

**验收标准**:
- **[EARS-017]** 当游戏进行中时，系统应显示剩余猜测次数。
- **[EARS-018]** 当游戏进行中时，系统应显示历史猜测记录及其颜色反馈。
- **[EARS-019]** 当游戏成功时，系统应显示成功提示和所用猜测次数。
- **[EARS-020]** 当游戏失败时，系统应显示失败提示和正确答案。

### 2.2 题库管理功能

#### 2.2.1 题目添加
**需求描述**: 管理员可以向题库添加新题目。

**验收标准**:
- **[EARS-021]** 当管理员添加单句题目时，系统应将题目保存到题库。
- **[EARS-022]** 当管理员批量添加题目时，系统应将所有有效题目保存到题库。
- **[EARS-023]** 当添加的题目已存在于题库时，系统应提示"题目已存在"。

#### 2.2.2 题目编辑
**需求描述**: 管理员可以编辑已有题目。

**验收标准**:
- **[EARS-024]** 当管理员选择编辑题目时，系统应显示题目编辑界面。
- **[EARS-025]** 当管理员修改题目内容并保存时，系统应更新题库中的对应题目。

#### 2.2.3 题目删除
**需求描述**: 管理员可以从题库删除题目。

**验收标准**:
- **[EARS-026]** 当管理员选择删除题目时，系统应从题库中移除该题目。
- **[EARS-027]** 当删除题目成功时，系统应提示"删除成功"。

#### 2.2.4 难度设置
**需求描述**: 管理员可以为题目设置难度等级。

**验收标准**:
- **[EARS-028]** 当管理员设置题目难度时，系统应保存难度信息。
- **[EARS-029]** 当难度设置成功时，系统应提示"难度设置成功"。

#### 2.2.5 分类管理
**需求描述**: 管理员可以创建和管理题目分类。

**验收标准**:
- **[EARS-030]** 当管理员创建新分类时，系统应保存分类信息。
- **[EARS-031]** 当管理员分配题目到分类时，系统应建立题目与分类的关联。
- **[EARS-032]** 当管理员从分类移除题目时，系统应解除题目与分类的关联。
- **[EARS-033]** 当删除分类时，系统应同时解除该分类下所有题目的关联。

---

### 2.3 诗词库管理功能

#### 2.3.1 诗词添加
**需求描述**: 管理员可以向诗词库添加新诗词。

**验收标准**:
- **[EARS-034]** 当管理员添加诗词时，系统应记录标题、作者、朝代、诗句等信息。
- **[EARS-035]** 当添加的诗词已存在时，系统应提示"诗词已存在"。

#### 2.3.2 诗词编辑
**需求描述**: 管理员可以编辑已有诗词。

**验收标准**:
- **[EARS-036]** 当管理员选择编辑诗词时，系统应显示诗词编辑界面。
- **[EARS-037]** 当管理员修改诗词信息并保存时，系统应更新诗词库中的对应记录。

#### 2.3.3 诗词删除
**需求描述**: 管理员可以从诗词库删除诗词。

**验收标准**:
- **[EARS-038]** 当管理员选择删除诗词时，系统应从诗词库中移除该记录。
- **[EARS-039]** 当删除诗词成功时，系统应提示"删除成功"。

#### 2.3.4 诗词查询
**需求描述**: 管理员可以查询诗词库中的诗词。

**验收标准**:
- **[EARS-040]** 当管理员输入搜索关键词时，系统应返回匹配的诗词列表。
- **[EARS-041]** 当搜索结果为空时，系统应提示"未找到匹配的诗词"。

---

### 2.4 数据导入导出功能

#### 2.4.1 数据导出
**需求描述**: 管理员可以导出题库和诗词库数据。

**验收标准**:
- **[EARS-042]** 当管理员选择导出数据时，系统应生成questions.json和poems.json文件。
- **[EARS-043]** 当导出成功时，系统应提示"导出成功"并显示文件保存路径。

#### 2.4.2 数据导入
**需求描述**: 管理员可以导入题库和诗词库数据。

**验收标准**:
- **[EARS-044]** 当管理员选择导入数据时，系统应读取并解析JSON文件。
- **[EARS-045]** 当导入文件格式错误时，系统应提示"文件格式错误"。
- **[EARS-046]** 当导入成功时，系统应提示"导入成功"并显示导入数量统计。

### 2.5 用户界面功能

#### 2.5.1 主界面
**需求描述**: 提供游戏主界面，包含游戏区域和操作按钮。

**验收标准**:
- **[EARS-047]** 当应用启动时，系统应显示游戏主界面。
- **[EARS-048]** 当显示主界面时，系统应包含输入框、猜测按钮、提示区域。
- **[EARS-049]** 当显示主界面时，系统应显示当前游戏状态信息。

#### 2.5.2 管理界面入口
**需求描述**: 提供进入管理界面的入口。

**验收标准**:
- **[EARS-050]** 当用户点击管理按钮时，系统应跳转到管理界面。
- **[EARS-051]** 当显示管理界面时，系统应包含题库管理和诗词库管理功能入口。

#### 2.5.3 响应式布局
**需求描述**: 界面应适配不同屏幕尺寸。

**验收标准**:
- **[EARS-052]** 当屏幕尺寸变化时，系统应自动调整界面布局。
- **[EARS-053]** 当设备为横屏时，系统应优化布局以充分利用屏幕空间。

## 3. 非功能需求

### 3.1 性能需求
- **[EARS-054]** 当玩家提交猜测时，系统应在500ms内完成匹配判断和反馈显示。
- **[EARS-055]** 当题库包含1000道题目时，系统应在100ms内完成随机选题。

### 3.2 可用性需求
- **[EARS-056]** 当玩家首次使用时，系统应提供游戏规则说明。
- **[EARS-057]** 当显示颜色反馈时，系统应同时提供颜色图例说明。

### 3.3 数据安全需求
- **[EARS-058]** 当保存数据时，系统应确保数据格式正确且完整。
- **[EARS-059]** 当应用异常退出时，系统应保证已保存的数据不丢失。

### 3.4 兼容性需求
- **[EARS-060]** 当运行于HarmonyOS设备时，系统应正常显示所有界面元素。
- **[EARS-061]** 当设备支持深色模式时，系统应适配深色模式显示。

## 4. 数据模型

### 4.1 题目（Question）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| content | string | 诗句内容 |
| poemId | string | 所属诗词ID |
| difficulty | number | 难度等级（1-5） |
| categoryIds | string[] | 所属分类ID列表 |

### 4.2 诗词（Poem）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| title | string | 诗词标题 |
| author | string | 作者 |
| dynasty | string | 朝代 |
| verses | string[] | 诗句列表 |

### 4.3 分类（Category）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| name | string | 分类名称 |
| description | string | 分类描述 |

### 4.4 猜测记录（GuessRecord）
| 字段 | 类型 | 说明 |
|------|------|------|
| content | string | 猜测内容 |
| feedback | FeedbackItem[] | 反馈结果列表 |
| timestamp | number | 猜测时间戳 |

### 4.5 反馈项（FeedbackItem）
| 字段 | 类型 | 说明 |
|------|------|------|
| char | string | 字符 |
| status | enum | 状态：correct/misplaced/absent |

---

## 5. 约束条件

### 5.1 业务约束
- 最大猜测次数固定为20次
- 题目答案必须存在于诗词库中
- 标点符号不参与匹配判断

### 5.2 技术约束
- 使用ArkTS语言开发
- 运行于HarmonyOS平台
- 使用ArkUI声明式开发范式
- 数据存储使用JSON格式

---

## 6. 排除范围

以下功能不在本期开发范围内：
- 用户账号系统
- 在线排行榜
- 多人对战模式
- 语音输入功能
- 诗词朗读功能
- 社交分享功能

---

## 7. 术语定义

| 术语 | 定义 |
|------|------|
| Wordle式反馈 | 一种颜色标记机制，用不同颜色表示字符匹配状态 |
| 题库 | 存储游戏题目的数据集合 |
| 诗词库 | 存储完整诗词信息的数据集合 |
| 上下句字数提示 | 显示答案诗句的上一句和下一句的字数 |
| ArkTS | HarmonyOS应用开发语言，基于TypeScript扩展 |
| ArkUI | HarmonyOS声明式UI开发框架 |
